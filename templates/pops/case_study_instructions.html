Case study details

Case study name

*Required*, field_type: string, maximum_length: 150 characters
Case study name is a required field that provides the user an opportunity to give the case study a descriptive name. The case study name will appear in a list of available case studies for you to choose from to start a session.
Example: Spotted Lanternfly, Pennsylvania, 2014-2022

Back to top

The first year that you have pest occurence data for calibration. This should correspond to the first year in the infestation data file.
Example: 2008

Final calibration year

Back to top
*Required*, field_type: integer, minimum_value: 1900, maximum_value: 2200
The last year that you have pest occurence data for calibration. This should correspond to the last year in the infestation data file.
Example: 2016

Final model year

Back to top
*Required*, field_type: integer, minimum_value: 1900, maximum_value: 2200
This will be the final year of a model run when using this case study. For example, if your final model year is 2024, then the model will run from the final calibration year to 2024.
Example: 2022

Time step

Back to top
*Required*, field_type: choice, options: Month, Week, Day
The time step is the frequency with which the PoPS model runs during a simulation year. The options for time step are: Month, Week, Day. A month time step will run much faster than a day time step because the model is running fewer iterations.
We recommend you choose the largest time step that is biologically relevant to your system. For example, if your insect has multiple generations in a year then set the timestep close to the length of the generation.
Example: Month

Score

Back to top
*Required*, field_type: decimal, minimum_value: 0.0, maximum_value: 1.0

Host score is a value between 0.0 and 1.0 that serves as an indication of the pest's (or pathogen's) preference for the host (or the host's ability to produce inoculum and become infected). Host score should be 1.0 in a single host system. In a multi-host system, host score provides a way to differentiate the pest's preference for the hosts.

For example, in a 2-host system, Host 1 could have a host score of 1.0, and Host 2 could have a host score of 0.5. This would indicated that the pest's preference for Host 1 is 2x's greater than its preference for Host 2 (or that Host 1 produces 2x's as much inoculum as Host 2).

Example: 0.87

Mortality rate method

Back to top

Temperature

Back to top
*Required*, field_type: boolean (on/off)
Does temperature affect the reproduction and survival of your pest/pathogen?

Temperature coefficient creation method

Back to top
*Conditionally Required* if Temperature is On, field_type: choice

If temperature affects the reproduction and survival of the pest/pathogen, then the PoPS model needs to include the effect of temperature in its calculations.

PoPS does this by using a "temperature coefficient" to modify reproductive and survival rates. Temperature values (in degrees Celsius) are transformed into a coefficient value between 0 and 1. A coefficient of "0.0" corresponds to no reproduction and survival. A coefficient of "1.0" corresponds to optimal reproduction and survival (i.e. ideal conditions for pest/pathogen spread).

For example, at extremely low temperatures and extremely high temperatures, we expect the coefficient to be close to 0.0 for most pests/pathogens. At the pest's ideal temperature, we expect the coefficient to be close to 1.0

There are two methods to transform raw temperature into a temperature coefficient used by the PoPS model:

  1. Reclass
  2. Polynomial

The method used is up to the user. Both methods are described below:

Reclass

Back to top
*Conditionally Required* if Temperature coefficient creation method is Reclass, field_type: decimal
If Reclass is selected as the Temperature coefficient creation method, then the user should provide the reclass coefficient for various temperature ranges. In the Reclass form, there is a column for:
  • Min temp: Minimum temperature (in Celsius) for this reclassification range.
  • Max temp: Maximum temperature (in Celsius) for this reclassification range.
  • Coefficient: Temperature coefficient for the temperature range indicated by Min temp and Max temp. Must be a value between 0 and 1. A coefficient of "0.0" corresponds to no reproduction and survival. A coefficient of "1.0" corresponds to optimal reproduction and survival (i.e. ideal conditions for pest/pathogen spread).
The user must provide at least 2 reclass rows, however, the user may add as many rows as is necessary to the Reclass matrix. The user is responsible for providing temperature coefficients for the full range of temperatures experienced by the pest/pathogen. Any temperatures not covered by the reclass matrix provided by the user will receive a temperature coefficient of "0.0".
Example:
Min temp Max temp Coefficient

Polynomial

Back to top
*Conditionally Required* if Temperature coefficient creation method is Polynomial, field_type: decimal

If Polynomial is selected as the Temperature coefficient creation method, then the user must provide the coefficients and constants for a polynomial equation to reclassify temperature as a coefficient between 0 and 1. A coefficient of "0.0" corresponds to no reproduction and survival. A coefficient of "1.0" corresponds to optimal reproduction and survival (i.e. ideal conditions for pest/pathogen spread).

The user should select the degree of the polynomial from the options (One, Two or Three). The number of coefficients and constants depend on the degree of the polynomial.

Polynomial degree options and their corresponding equations:

  • One: a0 + a1(T + x1)
  • Two: a0 + a1(T + x1) + a2(T + x2)2
  • Three: a0 + a1(T + x1) + a2(T + x2)2 + a3(T + x3)3

Where "T" is temperature (in degrees Celsius). an and xn must be provided by the user. The resulting polynomial should convert temperature (T) to a temperature coefficient ranging between 0 and 1. Any values less than 0.0 will be reclassed to 0.0. Any value greater than 1.0 will be reclassed to 1.0.

Example:
To use a polynomial of the form:

Coefficient = 0.6 + 0.02(T - 0.2) - 0.002(T - 10.0)2
