Case study details

Case study name

Back to top
*Required*, field_type: string, maximum_length: 150 characters
Case study name is a required field that provides the user an opportunity to give the case study a descriptive name. The case study name will appear in a list of available case studies for you to choose from to start a session.
Example: Spotted Lanternfly, Pennsylvania, 2014-2022

Initial calibration year

Back to top
*Required*, field_type: integer, minimum_value: 1900, maximum_value: 2200
The first year that you have pest occurence data for calibration. This should correspond to the first year in the all plants data file AND host data file.
Example: 2008

Final calibration year

Back to top
*Required*, field_type: integer, minimum_value: 1900, maximum_value: 2200
The last year that you have pest occurence data for calibration. This should correspond to the last year in the all plants data file AND host data file.
Example: 2016

Final model year

Back to top
*Required*, field_type: integer, minimum_value: 1900, maximum_value: 2200
This will be the final year of a model run when using this case study. For example, if your final model year is 2024, then the model will run from the final calibration year to 2024.
Example: 2016

Time step

Back to top
*Required*, field_type: choice, options: Month, Week, Year
The time step is the frequency with which the PoPS model is solved. The options for time step are: Month, Week, Day. In general, a day time step will be more accurate than a month time step, however a month time step will run much faster than a day time step because the model is solving fewer iterations.
We recommend you choose the largest time step possible for your case study, based on the biology of your system. For example, if ...
Example: Month

Number of pests

Back to top
*Required*, field_type: integer, minimum_value: 1
Number of pests (or pathogens) you will model in your system.
In the current version of PoPS, number of pests is limited to 1. Future versions will include multiple pests.
Example: 1

Number of hosts

Back to top
*Required*, field_type: integer, minimum_value: 1
Number of host organisms you will model in your system.
In the current version of PoPS, number of hosts is limited to 1. Future versions will include multiple hosts.
Example: 1

All plants

Back to top
*Required*, field_type: file field, format: raster stack
All plants is a raster stack (single file) of all plant data in your study area (1 layer for each year). At least 3 layers (3 years) are required for calibration and validation.
Example: my_plant_data.tiff

Infestation data

Back to top
*Required*, field_type: file field, format: raster stack
Infestation data is a raster stack (single file) of the initial infestation/infection data in your study area (1 layer for each year). At least 3 layers (3 years) are required for calibration and validation.
Example: initial_infestation.tiff

Use treatments

Back to top
*Required*, field_type: boolean (on/off)
Do you have existing management data to include for calibration/validation? Note: This field is for prior management/treatments. Only turn this field on if you have a raster file for management actions to include in the calibration and validation of the case study. You will still be able to use management during PoPS model runs if this is turned off.
Example: 1

Previous treatments data

Back to top
*Required if "Use treatments" is On*, field_type: file field, format: raster stack
Previous treatments data is a raster stack (single file) of the initial management data in your study area (1 layer for each year). At least 3 layers (3 years) are required for calibration and validation.
Example: previous_treatments.tiff
